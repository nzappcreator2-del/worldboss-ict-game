ให้พัฒนาระบบ **NPC ครูวีรภัทร์ ผู้มอบหมายภารกิจและรับส่งใบงาน** ในโปรเจกต์ NextGen Play โดยใช้ระบบ บทเรียน ใบงาน ผู้เล่น รางวัล และ Firestore ที่มีอยู่เดิมทั้งหมด ห้ามสร้างระบบใบงานใหม่ซ้ำ ห้ามรื้อ Logic เดิม และห้ามทำให้ Flow ปัจจุบันเสีย

## เป้าหมาย

เปลี่ยนประสบการณ์การเข้าถึงใบงานเดิม จากการกดเมนูหรือปุ่มธรรมดา ให้เป็นระบบรับภารกิจจาก NPC แบบเกม MMORPG

เมื่อนักเรียนเข้าสู่หน้าหลักของโหมดผจญภัยบทเรียน จะเห็น NPC 2D ชื่อ **“ครูวีรภัทร์”** ยืนอยู่ในพื้นที่หลัก นักเรียนสามารถเดินเข้าใกล้หรือกดที่ NPC เพื่อพูดคุย รับงาน เปิดใบงาน ส่งงาน และรับรางวัลผ่าน NPC ตัวนี้

ระบบใหม่นี้เป็นการเพิ่ม UX แบบเกมครอบระบบเดิม ไม่ใช่การเปลี่ยน Backend หรือสร้างระบบงานใหม่

## 1. เพิ่ม NPC ครูวีรภัทร์ในหน้าหลักผจญภัย

นำ NPC ครูวีรภัทร์ไปวางในบริเวณที่เห็นง่ายของหน้าหลัก Adventure Hub หรือหน้าหลักของโหมดผจญภัยบทเรียน

ใช้ Asset ตัวละคร 2D แบบโปร่งใส โดยเตรียมระบบให้สามารถเปลี่ยนไฟล์ Asset ภายหลังได้ง่าย หากยังไม่มี Asset จริง ให้ใช้ Placeholder ที่เข้ากับธีมเกมก่อน

NPC ต้องมีองค์ประกอบดังนี้

* ชื่อเหนือศีรษะ “ครูวีรภัทร์”
* คำอธิบายสั้น เช่น “ผู้มอบหมายภารกิจ”
* Idle Animation เช่น ขยับตัวเบาๆ กระพริบตา หรือเสื้อผ้าขยับ
* วงแสงหรือเงาใต้เท้า
* เอฟเฟกต์เมื่อมีภารกิจใหม่
* รองรับการกดด้วยเมาส์และการแตะบนมือถือ
* หากหน้าหลักมีระบบเดินตัวละคร ให้เปิดการโต้ตอบเมื่อผู้เล่นเดินเข้าใกล้
* หากยังไม่มีระบบเดินในหน้าดังกล่าว ให้ผู้เล่นสามารถกด NPC ได้โดยตรง

## 2. สถานะเหนือศีรษะ NPC

ให้ NPC แสดง Marker ตามสถานะของใบงานหรือภารกิจปัจจุบัน โดยอ่านจากข้อมูลระบบเดิม

### มีงานใหม่

แสดงเครื่องหมาย `!` สีทองหรือสีเหลืองเหนือศีรษะ

เอฟเฟกต์ควรมี

* ลอยขึ้นลงเบาๆ
* Pulse หรือ Glow
* มีประกายเล็กน้อย
* เด่นแต่ไม่รบกวนสายตา
* อาจมีข้อความลอยเป็นครั้งคราวว่า “มีภารกิจใหม่!”

### รับงานแล้วแต่ยังทำไม่เสร็จ

เปลี่ยนเป็นเครื่องหมาย `…` หรือไอคอนสมุดงาน

แสดงว่าผู้เล่นมีภารกิจที่กำลังดำเนินการ

### ทำงานเสร็จและพร้อมส่ง

แสดงเครื่องหมาย `?` สีเขียวหรือสีทอง

เพิ่มเอฟเฟกต์ให้เด่นกว่าสถานะอื่น เพื่อบอกเด็กว่าต้องกลับมาส่งงานกับ NPC

### ส่งงานสำเร็จแล้ว

นำ Marker ออก หรือเปลี่ยนเป็นเครื่องหมายถูกขนาดเล็กชั่วคราว

หลังจากแสดงผลสำเร็จแล้ว ให้ NPC กลับสู่สถานะปกติ หรือแสดงภารกิจถัดไปหากมี

## 3. การพูดคุยกับ NPC

เมื่อผู้เล่นกด NPC หรือเข้าใกล้แล้วกดปุ่ม “พูดคุย” ให้เปิดหน้าต่าง Dialogue แบบเกม RPG

หน้าต่างควรมี

* Portrait หรือภาพครึ่งตัวของครูวีรภัทร์
* ชื่อ “ครูวีรภัทร์”
* ข้อความสนทนา
* ชื่อภารกิจ
* สถานะภารกิจ
* ปุ่มการทำงานตามสถานะ

ใช้ข้อความที่เข้าใจง่าย เหมาะกับนักเรียน และมีบรรยากาศแบบเกมผจญภัย

ตัวอย่างข้อความเมื่อมีงานใหม่

“สวัสดีนักผจญภัย วันนี้ครูมีภารกิจใหม่ให้เธอ ลองศึกษาบทเรียนและทำใบงานนี้ให้สำเร็จนะ”

ปุ่มที่แสดง

* รับภารกิจ
* ดูรายละเอียด
* ไว้ก่อน

ตัวอย่างข้อความเมื่อกำลังทำงาน

“ภารกิจเป็นอย่างไรบ้าง หากไม่แน่ใจสามารถกลับไปดูบทเรียนได้เสมอนะ”

ปุ่มที่แสดง

* ทำภารกิจต่อ
* ดูรายละเอียดงาน
* กลับไปยังบทเรียน
* ปิด

ตัวอย่างข้อความเมื่อพร้อมส่ง

“ยอดเยี่ยมมาก ครูเห็นว่าเธอทำภารกิจเรียบร้อยแล้ว พร้อมส่งงานหรือยัง?”

ปุ่มที่แสดง

* ส่งงาน
* ตรวจคำตอบอีกครั้ง
* ปิด

ตัวอย่างข้อความเมื่อส่งสำเร็จ

“ทำได้ดีมาก ภารกิจนี้สำเร็จแล้ว ครูมอบรางวัลให้กับความพยายามของเธอ!”

## 4. การเชื่อมกับระบบใบงานเดิม

ใช้ระบบใบงานและการบันทึกคำตอบเดิมที่มีอยู่แล้ว

ห้ามสร้าง Collection หรือข้อมูลใบงานชุดใหม่ซ้ำโดยไม่จำเป็น

ให้ NPC ทำหน้าที่เป็นทางเข้าสู่ระบบเดิมดังนี้

* เมื่อรับภารกิจ ให้บันทึกเฉพาะสถานะสำหรับ UX หากระบบเดิมยังไม่มีสถานะดังกล่าว
* ปุ่ม “เริ่มทำภารกิจ” ให้เปิดหน้าใบงานเดิมของบทเรียนที่เกี่ยวข้อง
* ปุ่ม “ทำภารกิจต่อ” ให้กลับไปยังใบงานเดิมพร้อมข้อมูลที่ผู้เล่นเคยกรอก
* ปุ่ม “ส่งงาน” ให้เรียกใช้ Flow การส่งใบงานเดิม
* ใช้เงื่อนไขการได้รับ XP และ Coins เดิม
* ห้ามให้รางวัลซ้ำ หากผู้เล่นส่งหรือเปิดภารกิจเดิมหลายครั้ง
* คำตอบและข้อมูลการส่งงานต้องยังคงอยู่หลัง Refresh หรือกลับเข้าระบบใหม่ตามพฤติกรรมเดิมของโปรเจกต์

หากใบงานเดิมบันทึกและให้รางวัลทันทีตอนผู้เล่นกดส่ง ให้ปรับประสบการณ์ดังนี้

1. ผู้เล่นกรอกและยืนยันคำตอบในหน้าใบงาน
2. ระบบบันทึกคำตอบตามเดิม
3. เปลี่ยนสถานะ NPC เป็น “พร้อมส่งภารกิจ”
4. ผู้เล่นกลับมาคุยกับครูวีรภัทร์
5. NPC แสดงผลสำเร็จและรางวัลที่ได้รับแล้ว

ต้องไม่จ่ายรางวัลซ้ำในขั้นตอนที่ 5 หากระบบเดิมจ่ายรางวัลไปแล้ว ขั้นตอนนี้เป็นเพียงการนำเสนอผลรางวัลผ่าน NPC

## 5. Quest Detail

เมื่อกด “ดูรายละเอียด” ให้เปิด Quest Card หรือ Quest Panel โดยดึงข้อมูลจากบทเรียนและใบงานเดิม

แสดงข้อมูลดังนี้

* ชื่อภารกิจ
* ชื่อบทเรียน
* ผู้มอบหมาย: ครูวีรภัทร์
* คำอธิบายภารกิจ
* เป้าหมาย
* ความคืบหน้า
* รางวัล
* สถานะปัจจุบัน

ตัวอย่างเป้าหมาย

* ศึกษาบทเรียน 1 ครั้ง
* ทำใบงานให้ครบ
* ส่งงานให้ครูวีรภัทร์

ตัวอย่างสถานะ

* ภารกิจใหม่
* กำลังดำเนินการ
* พร้อมส่ง
* สำเร็จแล้ว

Quest Card ต้องใช้ข้อมูลจริงจากระบบเดิม ห้ามใส่รางวัลหรือความคืบหน้าแบบหลอก หากข้อมูลใดไม่มีให้ซ่อนส่วนนั้นอย่างเหมาะสม

## 6. Quest Tracker

หลังจากผู้เล่นรับภารกิจ ให้มี Quest Tracker ขนาดกะทัดรัดอยู่ด้านข้างหน้าจอในโหมดผจญภัย

แสดง

* ชื่อภารกิจที่กำลังติดตาม
* เป้าหมายปัจจุบัน
* ความคืบหน้า
* คำแนะนำว่าต้องทำอะไรต่อ

ตัวอย่าง

“ภารกิจ: ความปลอดภัยบนโลกออนไลน์
ศึกษาบทเรียน 1/1
ทำใบงาน 0/1”

เมื่อทำใบงานเสร็จ ให้เปลี่ยนข้อความเป็น

“กลับไปส่งงานกับครูวีรภัทร์”

การกด Quest Tracker สามารถเปิดรายละเอียดภารกิจหรือพาผู้เล่นกลับไปยัง NPC ได้ตามความเหมาะสมของหน้าปัจจุบัน

## 7. การแจ้งเตือนเมื่อมีงานใหม่

เมื่อผู้เล่นเข้าสู่ Adventure Hub และมีใบงานใหม่ ให้แสดง Notification แบบเกมช่วงสั้นๆ

ตัวอย่าง

“มีภารกิจใหม่จากครูวีรภัทร์”

การแจ้งเตือนต้อง

* ไม่แสดงซ้ำถี่เกินไป
* ไม่บังหน้าจอหลัก
* ปิดเองได้
* กดแล้วเลื่อนไปยัง NPC หรือเปิด Dialogue ได้
* รองรับมือถือ

## 8. หน้าจอภารกิจสำเร็จ

เมื่อผู้เล่นกลับมาส่งงานกับ NPC ให้แสดง Quest Complete Overlay แบบเกม MMORPG

แสดง

* ข้อความ “ภารกิจสำเร็จ!”
* ชื่อภารกิจ
* XP ที่ได้รับ
* Coins ที่ได้รับ
* Badge หรือ Item หากมีจริง
* ปุ่ม “รับรางวัล” หรือ “เสร็จสิ้น”

เพิ่ม Feedback เช่น

* แสงประกาย
* XP และ Coins เด้งขึ้น
* เสียงสำเร็จแบบสั้น
* NPC แสดง Animation ดีใจหรือปรบมือ
* Marker เหนือศีรษะเปลี่ยนสถานะทันที

ใช้ค่ารางวัลจากผลลัพธ์ของระบบเดิมเท่านั้น ห้ามคำนวณรางวัลชุดใหม่ซ้ำ

## 9. พฤติกรรม NPC เพิ่มเติม

เพื่อให้ NPC ดูมีชีวิต ให้เพิ่มข้อความพูดลอยแบบสุ่มในช่วงเวลาที่เหมาะสม เช่น

* “มีภารกิจใหม่รออยู่นะ”
* “อย่าลืมกลับมาส่งงานกับครู”
* “ความพยายามทำให้เราเก่งขึ้น”
* “ติดตรงไหน กลับไปทบทวนบทเรียนได้นะ”
* “ทำได้ดีมาก เหลืออีกนิดเดียว!”

อย่าแสดงข้อความถี่หรือซ้ำจนรบกวนผู้เล่น

เมื่อผู้เล่นเดินผ่านหรือเปิดหน้า Adventure Hub อาจให้ NPC โบกมือหรือหันมาทางผู้เล่นหนึ่งครั้ง

## 10. รูปแบบภาพและ UX

ออกแบบให้เข้ากับธีม MMORPG/RPG เดิมของ NextGen Play

ใช้รูปแบบเดียวกับ UI ที่มีอยู่ เช่น

* Typography
* ปุ่ม
* Panel
* Border
* Glow
* Shadow
* Animation
* Responsive behavior

อย่าสร้าง UI ที่ดูเป็นระบบเว็บไซต์โรงเรียนหรือแบบฟอร์มทั่วไป

หน้าต่าง NPC และ Quest ต้องให้ความรู้สึกว่าเป็นส่วนหนึ่งของโลกเกม แต่ยังอ่านง่ายสำหรับนักเรียนระดับประถม

รองรับทั้ง

* Desktop
* Tablet
* Mobile
* เมาส์
* Touch screen
* การแสดงผลแนวตั้งและแนวนอนตามพื้นที่ที่ระบบเดิมรองรับ

## 11. ความปลอดภัยและความถูกต้อง

* อย่าเชื่อข้อมูลรางวัลจาก UI เพียงอย่างเดียว
* ใช้ Service และ Firestore Rules เดิม
* ห้ามเปิดสิทธิ์อ่านหรือเขียนข้อมูลนักเรียนเพิ่มเติมโดยไม่จำเป็น
* ห้ามเปลี่ยนระบบ Owner UID
* ห้ามทำให้ผู้เล่นสามารถรับรางวัลซ้ำด้วยการ Refresh หรือเปิด Dialogue หลายครั้ง
* ห้ามบันทึกข้อมูลส่วนตัวลงใน Quest Tracker หรือพื้นที่สาธารณะ
* Error ทุกจุดต้องมีข้อความภาษาไทยที่เข้าใจง่าย
* หากโหลดข้อมูลไม่ได้ ให้ NPC ยังแสดงได้ แต่แจ้งว่า “ยังโหลดภารกิจไม่สำเร็จ กรุณาลองใหม่”

## 12. สิ่งที่ห้ามทำ

* ห้ามรื้อระบบใบงานเดิม
* ห้ามสร้างระบบ Login ใหม่
* ห้ามสร้างระบบรางวัลใหม่ซ้ำ
* ห้ามสร้าง Firestore Collection ใหม่จำนวนมากโดยไม่จำเป็น
* ห้ามเปลี่ยนการคำนวณ XP, Coins, Level หรือ Rank เดิม
* ห้ามลบปุ่มหรือทางเข้าระบบเดิมทันที จนกว่าจะยืนยันว่า NPC Flow ทำงานครบ
* ห้ามทำ NPC เป็นเพียงภาพนิ่งพร้อมปุ่มธรรมดา
* ห้ามใช้ Popup แบบเว็บทั่วไปที่ไม่เข้ากับธีมเกม
* ห้ามทำ Animation หนักจนกระทบเครื่องโรงเรียนหรือโทรศัพท์สเปกต่ำ

## 13. ลำดับการทำงานที่ต้องการ

1. ตรวจระบบใบงานเดิมและ Flow การส่งงานปัจจุบัน
2. ระบุสถานะที่สามารถนำมาใช้เป็น Quest State ได้
3. เพิ่ม NPC ครูวีรภัทร์ในหน้า Adventure Hub
4. เพิ่ม Marker และ Animation ตามสถานะ
5. เพิ่ม Dialogue และ Quest Detail
6. เชื่อมปุ่มเริ่มงานและส่งงานเข้ากับระบบเดิม
7. เพิ่ม Quest Tracker
8. เพิ่ม Notification เมื่อมีงานใหม่
9. เพิ่ม Quest Complete Overlay โดยใช้ผลรางวัลเดิม
10. ทดสอบ Refresh, Login ใหม่, ส่งซ้ำ, งานที่ทำแล้ว และกรณี Network Error
11. ตรวจ Responsive และประสิทธิภาพบนมือถือ/เครื่องสเปกต่ำ
12. รัน lint, typecheck, tests และ build ตาม Workflow เดิมของโปรเจกต์

## 14. เกณฑ์ถือว่างานเสร็จ

ระบบถือว่าเสร็จเมื่อ

* ผู้เล่นเห็น NPC ครูวีรภัทร์ในหน้า Adventure Hub
* Marker เปลี่ยนตามสถานะงานจริง
* ผู้เล่นรับภารกิจจาก NPC ได้
* เปิดใบงานเดิมผ่าน NPC ได้
* ทำงานต่อจากข้อมูลเดิมได้
* ส่งงานผ่าน Flow เดิมได้
* กลับมารับผลสำเร็จกับ NPC ได้
* ไม่ได้รับ XP หรือ Coins ซ้ำ
* Quest Tracker อัปเดตถูกต้อง
* Refresh แล้วสถานะไม่หายหรือย้อนผิด
* ใช้งานได้ทั้ง Desktop และ Mobile
* ไม่ทำให้หน้า Lesson, Worksheet, Dashboard, PVP, World Boss และ Admin เดิมเสีย
* Tests และ Build ผ่านทั้งหมด

หลังพัฒนาเสร็จ ให้สรุปว่าแก้ไฟล์ใดบ้าง ระบบใหม่เชื่อมกับระบบเดิมตรงไหน มีข้อจำกัดใดเหลืออยู่ และวิธีทดสอบ Flow ตั้งแต่รับภารกิจจนถึงส่งงานสำเร็จ
